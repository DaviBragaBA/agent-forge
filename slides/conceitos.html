4 tipos de agent

Modo de operação — como o agent recebe e processa pedidos.

⚡

Task-Based

Recebe tarefa bem definida, executa e retorna. Fluxo direto sem decomposição.

Ideal
Entrada clara · Objetivo específico · Sem ambiguidade
Ex.
MonitorAgent: alerta → diagnóstico
💬

Interativo

Entrada ambígua — faz perguntas antes de agir.

Ideal
Contexto incompleto · Decisões críticas · Evitar suposições
Ex.
Tutor: lacunas → pergunta → explica
🎯

Orientado a Objetivo

Objetivo amplo → decompõe em sub-objetivos executáveis.

Ideal
Problemas complexos · Planejamento · Múltiplas etapas
Ex.
Backlog: requisito → épicos → histórias
🤖

Autônomo

Reage a eventos. Não depende de pedido humano direto.

Ideal
Monitoramento · Triggers · Operação 24/7
Ex.
Watchdog: anomalia → ação limitada
⚠ Ações sensíveis EXIGEM confirmação humana

// arquiteturas

3 arquiteturas cognitivas

Como o agent pensa — definido em planner.md.

🔄

ReAct

Pensa, age e observa em ciclo. Descobre o caminho enquanto avança.

+
Adapta-se · Incerteza
−
Alto consumo de tokens · Menos previsível
Ideal
Problemas exploratórios, caminho incerto
📋

Plan-and-Execute

Plano completo no início, executa passo a passo.

+
Previsível · Eficiente · Auditável
−
Não adapta · Falha silenciosa se cenário muda
Ideal
Processos estruturados, etapas conhecidas
🪞

Reflection

Executa, avalia o próprio resultado e refina antes de finalizar.

+
Alta qualidade · Detecta erros antes de entregar
−
Maior latência · Maior custo
Ideal
Qualidade crítica, erro caro

4 adapters

Conexão com o mundo real — dados, APIs e ferramentas.

🎭

Mock

Dados simulados via LLM. Prototipação e testes.

Uso
Aprendizado, desenvolvimento inicial
🌐

REST

Chamadas HTTP a APIs reais.

Uso
Integração com serviços externos
🗄️

Database

Queries em banco (preferencialmente read-only e parametrizadas).

Uso
Consulta de dados estruturados
🔒 Apenas leitura · Queries parametrizadas · Validação rigorosa
🔌

MCP

Model Context Protocol — comunicação padronizada com ferramentas.

Uso
Integração padronizada e reutilizável
