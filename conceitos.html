// tipos

4 tipos de agent

Modo de operação — como o agent recebe e processa pedidos.

⚡

Task-Based

Recebe tarefa bem definida, executa e retorna. Fluxo direto sem decomposição.

IdealEntrada clara · Objetivo específico · Sem ambiguidade
Ex.MonitorAgent: recebe alerta, retorna diagnóstico
💬

Interativo

Quando a entrada é ambígua, faz perguntas antes de agir.

IdealContexto incompleto · Decisões críticas · Evitar suposições
Ex.Tutor: identifica lacunas e pergunta antes de explicar
🎯

Orientado a Objetivo

Recebe objetivo amplo, decompõe em sub-objetivos executáveis.

IdealProblemas complexos · Planejamento estruturado · Múltiplas etapas
Ex.Backlog Decomposer: requisito → épicos → histórias → critérios
🤖

Autônomo

Reage a eventos. Não depende de solicitação humana direta.

IdealMonitoramento contínuo · Resposta a triggers · Operação 24/7
Ex.Watchdog: detecta anomalia e age dentro de limites
⚠ Ações sensíveis EXIGEM confirmação humana
// arquiteturas

3 arquiteturas cognitivas

Como o agent pensa — definido em planner.md.

🔄

ReAct

Pensa, age e observa em ciclo. Descobre o caminho enquanto avança.

+Adapta-se ao contexto · Funciona bem com incerteza
−Alto consumo de tokens · Menos previsível
IdealProblemas exploratórios, caminho incerto
📋

Plan-and-Execute

Cria plano completo no início e executa passo a passo.

+Previsível · Eficiente · Auditável
−Não se adapta a mudanças · Falha silenciosamente se cenário muda
IdealProcessos estruturados, etapas conhecidas
🪞

Reflection

Executa, avalia o próprio resultado e refina antes de finalizar.

+Alta qualidade · Detecta erros antes de entregar
−Maior latência · Maior custo
IdealCenários onde qualidade é crítica e erro é caro

4 adapters

Conexão com o mundo real — dados, APIs e ferramentas.

🎭

Mock

Dados simulados via LLM. Para prototipação e testes.

UsoAprendizado, desenvolvimento inicial
🌐

REST

Chamadas HTTP a APIs reais.

UsoIntegração com serviços externos
🗄️

Database

Queries em banco de dados (preferencialmente read-only e parametrizadas).

UsoConsulta de dados estruturados
🔒 Apenas leitura · Queries parametrizadas · Validação rigorosa
🔌

MCP (Model Context Protocol)

Protocolo padronizado de comunicação com ferramentas.

UsoIntegração padronizada e reutilizável
